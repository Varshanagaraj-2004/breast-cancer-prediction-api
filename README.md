# Breast Cancer Disease Prediction API

Predicts whether a breast tumor is **Benign** or **Malignant** from digitized
fine-needle-aspirate (FNA) cell measurements, using the classic
[UCI Wisconsin Breast Cancer dataset](https://archive.ics.uci.edu/dataset/17/breast+cancer+wisconsin+diagnostic).

The project trains and compares four classifiers — **Logistic Regression, SVM,
Random Forest, and XGBoost** — then serves the best one through a **FastAPI**
REST API, ready for a demo video and GitHub upload.

## Results

| Model               | CV Accuracy | Test Accuracy | Precision | Recall | F1     | ROC-AUC |
|---------------------|:-----------:|:--------------:|:---------:|:------:|:------:|:-------:|
| Logistic Regression | 0.9736      | 0.9649         | 0.9750    | 0.9286 | 0.9512 | 0.9960  |
| **SVM (RBF)** ⭐      | 0.9714      | **0.9737**     | **1.0000**| 0.9286 | **0.9630** | 0.9947  |
| Random Forest        | 0.9604      | 0.9737         | 1.0000    | 0.9286 | 0.9630 | 0.9944  |
| XGBoost              | 0.9714      | 0.9649         | 1.0000    | 0.9048 | 0.9500 | 0.9937  |

**SVM (RBF kernel)** was selected as the serving model (highest F1 / test
accuracy, perfect precision — zero false "malignant" alarms on the held-out
test set). Full metrics are also available live at the `/model-info` endpoint.

## Setup

```bash
git clone <your-repo-url>
cd breast-cancer-api
python3 -m venv venv && source venv/bin/activate
pip install -r requirements.txt
```

## 1. Train the models

```bash
python train.py
```

This prints a comparison table for all four algorithms and saves the
best-performing model + scaler into `models/`.

## 2. Run the API

```bash
uvicorn app.main:app --reload --port 8000
```

Open **http://127.0.0.1:8000/docs** for the interactive Swagger UI — this is
the easiest way to record a demo video: you can trigger real predictions
directly from the browser.

## Endpoints

| Method | Path              | Description                                             |
|--------|-------------------|----------------------------------------------------------|
| GET    | `/`               | API status + which model is being served                 |
| GET    | `/health`         | Health check                                              |
| GET    | `/model-info`     | Full metrics for every model that was compared            |
| POST   | `/predict`        | Predict from 30 tumor measurements (see example below)    |
| POST   | `/predict/sample` | Runs a prediction on a built-in example (no body needed)  |

## Tips for the demo video

1. Start the server: `uvicorn app.main:app --reload --port 8000`
2. Open `http://127.0.0.1:8000/docs` in your browser.
3. Expand `POST /predict/sample`, click **Try it out → Execute** — shows an
   instant malignant prediction with probabilities.
4. Expand `GET /model-info` to show the side-by-side metrics for all four
   algorithms — good talking point for "why SVM was chosen."
5. Optionally show `POST /predict` with your own values to prove it generalizes.

## Dataset

Wisconsin Diagnostic Breast Cancer (WDBC) dataset — 569 samples, 30 numeric
features computed from digitized images of a fine needle aspirate (FNA) of a
breast mass, describing characteristics of the cell nuclei present in the image.
Target: `diagnosis` — Malignant (M) or Benign (B).

## Disclaimer

This project is for educational / demo purposes only and is **not** a
medical device. Predictions should never be used for real clinical
decision-making.
