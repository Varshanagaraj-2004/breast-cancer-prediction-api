# 🩺 Breast Cancer Diagnosis Prediction API

![Python](https://img.shields.io/badge/Python-3.10%2B-blue)
![FastAPI](https://img.shields.io/badge/FastAPI-REST%20API-009688)
![scikit--learn](https://img.shields.io/badge/scikit--learn-ML-orange)
![License](https://img.shields.io/badge/License-MIT-lightgrey)

A machine learning REST API that classifies a breast tumor as **Benign** or
**Malignant** from digitized fine-needle-aspirate (FNA) cell measurements,
using the classic [UCI Wisconsin Breast Cancer dataset](https://archive.ics.uci.edu/dataset/17/breast+cancer+wisconsin+diagnostic).

Four classifiers — **Logistic Regression, SVM, Random Forest, and XGBoost** —
are trained and compared; the best-performing one is served through a
**FastAPI** backend with interactive Swagger documentation.

> **What this is:** a classifier that scores an *already-biopsied* tissue
> sample's malignancy probability from its cell measurements.
> **What this is not:** a screening tool that scans mammograms/ultrasounds to
> find tumors from scratch. See [Scope & Disclaimer](#-scope--disclaimer).

## 📊 Model comparison

| Model               | CV Accuracy | Test Accuracy | Precision  | Recall | F1         | ROC-AUC |
|---------------------|:-----------:|:--------------:|:----------:|:------:|:----------:|:-------:|
| Logistic Regression | 0.9736      | 0.9649         | 0.9750     | 0.9286 | 0.9512     | 0.9960  |
| **SVM (RBF)** ⭐      | 0.9714      | **0.9737**     | **1.0000** | 0.9286 | **0.9630** | 0.9947  |
| Random Forest        | 0.9604      | 0.9737         | 1.0000     | 0.9286 | 0.9630     | 0.9944  |
| XGBoost              | 0.9714      | 0.9649         | 1.0000     | 0.9048 | 0.9500     | 0.9937  |

**SVM (RBF kernel)** was selected as the serving model — highest F1 and test
accuracy, with perfect precision (zero false "malignant" alarms on the held-out
test set). All metrics above are also available live at the `/model-info`
endpoint, generated fresh each time the models are trained.

## 🚀 Getting started

```bash
git clone https://github.com/Varshanagaraj-2004/breast-cancer-prediction-api.git
cd breast-cancer-prediction-api
python -m venv venv
venv\Scripts\activate          # Windows
# source venv/bin/activate     # macOS / Linux
pip install -r requirements.txt
```

### 1. Train the models (optional — trained models are already committed)

```bash
python train.py
```

Prints a comparison table for all four algorithms and saves the
best-performing model + scaler into `models/`.

### 2. Run the API

```bash
python -m uvicorn app.main:app --reload --port 8000
```

Open **http://127.0.0.1:8000/docs** for the interactive Swagger UI — the
easiest way to try predictions directly from the browser, no coding required.

## 📡 Endpoints

| Method | Path              | Description                                              |
|--------|-------------------|------------------------------------------------------------|
| GET    | `/`               | API status + which model is being served                   |
| GET    | `/health`         | Health check                                                |
| GET    | `/model-info`     | Full metrics for every model that was compared              |
| POST   | `/predict`        | Predict from 30 tumor measurements (see example below)      |
| POST   | `/predict/sample` | Runs a prediction on a built-in example — no input needed   |

## 🎥 Demo walkthrough

1. Start the server: `python -m uvicorn app.main:app --reload --port 8000`
2. Open `http://127.0.0.1:8000/docs` in your browser.
3. Expand **POST /predict/sample** → **Try it out** → **Execute** — instant
   prediction with probabilities, zero setup.
4. Expand **GET /model-info** → **Try it out** → **Execute** — shows the
   side-by-side metrics for all four algorithms (good talking point for "why
   SVM was chosen").
5. Optionally, expand **POST /predict** and paste in custom values to show it
   generalizing to new data.

## 🧬 Dataset

Wisconsin Diagnostic Breast Cancer (WDBC) dataset — 569 samples, 30 numeric
features computed from a digitized image of a fine needle aspirate (FNA) of a
breast mass, describing characteristics of the cell nuclei present in the
image (radius, texture, perimeter, smoothness, concavity, etc.).
Target: `diagnosis` — Malignant (M) or Benign (B).

## ⚠️ Scope & disclaimer

This model predicts malignancy **from measurements already extracted from a
biopsied tissue sample** — it does not analyze mammograms, ultrasounds, or
any raw medical imaging, and it cannot screen a patient for cancer from
scratch. It is a **classification** system, not a **detection/screening**
system.

This project is for educational and portfolio purposes only. It is **not** a
certified medical device and must never be used for real clinical
decision-making.
